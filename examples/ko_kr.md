## 1. 수식 표현
<a name="ch1-1"></a>
### 사칙 연산
연관 파일 : `ch1/add.mp`\
연산은 사칙연산, 비교연산, 행렬연산, 지수연산, 범위연산을 지원합니다.
먼저, 간단한 사칙연산을 구현해봅시다.
```
@ a = 3
@ b = 4
@ c = a + b
@ 
@ print c

c = 7
```
<a name="ch1-2"></a>
### 비교문 수식
연관 파일 : `ch1/comparison.mp`\
간단한 비교연산을 구현해봅시다.
* 본 연산은 실험중입니다.
```
@ a = 3
@ b = 4
@ c = a < b
@ 
@ print c

c = True
```
<a name="ch1-3"></a>
### 복잡한 수식
연관 파일 : `ch1/expression.mp`\
복잡한 수식을 간단히 표현할 수 있습니다.
```
@ a = 1
@ b = -2
@ c = 1
@ 
@ D = b ** 2 - 4 * a * c
@ x = (-1 * b - D ** .5) / (2 * a)
@ 
@ print D, x

D = 0
x = 1.0
```
<a name="ch1-4"></a>
### 일회성 정의
연관 파일 : `ch1/disposable.mp`\
`:=` 연산자는 변수에 값이 이미 할당돼있는 경우, 값을 대입하지 않습니다.
특히, 변수에 해당하는 파일이 존재하면 대입이 이루어지지 않습니다.
다만, 적어도 한번이라도 변수에 다음과 같은 대입을 시도한다면,
`save` 시에 해당 변수의 대입 연산자는 `=`에서 `:=`로 바뀝니다.
```
@ # a에는 값이 있으므로 대입이 이루어지지 않는다.
@ a = 1
@ a := 2
@ 
@ # b에는 값이 없으므로 대입이 이루어진다.
@ b := 1
@ b = 2
@ 
@ print a, b

a = 1
b = 2
```

<a name="ch2"></a>
## 2. 메소드 호출
<a name="ch2-1"></a>
### 내장 메소드 호출
연관 파일 : `ch2/method.mp`\
내장 메소드를 호출해봅시다. 현재 `tensor`, `max`, `min`이 구현되었습니다.
```
@ # 함수를 가리키는 변수들
@ foo = tensor
@ goo = foo
@ too = goo
@ 
@ # (3, 4) 크기의 배열을 만들고, 2를 더한다.
@ potato = too(3, 4) + 2
@ # 배열을 파이썬처럼(Pythonic) 인덱싱할 수 있습니다.
@ k = potato(0, 1:3) + 3
@ 
@ print potato, k

k = [5. 5.]
potato = [[2. 2. 2. 2.]
 [2. 2. 2. 2.]
 [2. 2. 2. 2.]]
```

<a name="ch3"></a>
## 3. 사용자 정의 메소드 호출
<a name="ch3-1"></a>
### 수식 불러오기
연관 파일 : `ch3/saved.mp`, `ch3/aaa.mp`\
`save` 접두사를 이용하면 변수가 **파일**로 저장됩니다.
덕분에 `MP`를 다시 호출하여 변수를 쉽게 불러올 수 있습니다.
```
# 인터프리터 모드 (aaa.mp 파일 생성)
@ x = 3
@ y = 4
@ aaa = x + y
@ save aaa
```
```
# 생성된 aaa.mp
(aaa=((x=3)+(y=4)))
```
```
# 인터프리터 모드 (aaa.mp 파일 불러오기)
@ print aaa

aaa = 7
```
<a name="ch3-2"></a>
### 함수 구현하기
연관 파일 : `ch3/def.mp`\
`def` 메소드를 이용하여 함수(매크로)를 구현할 수 있습니다.
```
@ foo = def(x, y, x + y)
@ aaa = foo(3, 4)
@ print aaa

aaa = 7
```
<a name="ch3-3"></a>
### 피드백 구현하기
연관 파일 : `ch3/feed.mp`\
반복문을 통해 함수의 출력값이 다시 함수의 입력값으로 사용되는 구조를 만들 수 있습니다.
`mp`에서는 마지막 인자를 `Feedback Argument`라고 정의합니다.
이는, 두 번째 반복부터는 입력값 대신 이전의 출력값을 재사용한다는 뜻입니다.
예시는 `3`을 `5`번 곱하는 알고리즘입니다.
```
@ # 두 수의 곱
@ mul = def(mul_x, mul_y, mul_x * mul_y)
@ 
@ # 피드백 구조를 구축합니다.
@ pow_z = (mul * pow_y)(pow_x, 1)
@ pow = def(pow_x, pow_y, pow_z)
@ 
@ # 실험값
@ a = mul(3, 5)
@ b = pow(3, 5)
@ print a, b

a = 15
b = 243
```

## 4. 긴 문자열 변수
<a name="ch4-1"></a>
### 문장 형식의 변수명
연관 파일 : `ch4/string.mp`\
`MP`는 문자열을 지원하지 않습니다. 대신 변수명을 문자열처럼 사용 가능합니다.
문자열에 띄워쓰기와 `'`, `"`, `_`, `^` 등의 특수기호를 허용합니다.
단, 띄워쓰기는 변수명에서 무시됩니다.
```
@ my house = 3
@ my cat's year = 2.9
@ 
@ # my house는 myhouse와 같습니다.
@ print myhouse, my cat's year

myhouse = 3
mycat'syear = 2.9
```
<a name="ch4-2"></a>
### 디렉토리 형식의 변수명
연관 파일 : `ch4/recursion.mp`\
만약 새로운 디렉토리를 만들고, 그곳에 변수를 저장하고 싶다면,
간단히 `.`을 이용하여 디렉토리 구별이 가능합니다.
디렉토리는 자동으로 만들어지고, 빈 디렉토리는 자동으로 제거됩니다.
* 본 연산은 실험중입니다.
```
@ a.b = 3 + 4
@ save a.b
@ 
@ c = 5
@ from a save c

(a 디렉토리에 b.mp 파일 생성)
(a 디렉토리에 c.mp 파일 생성)
```

## 6. SSH를 통한 원격 인터프리터 실행
<a name="ch6-1"></a>
### 원격 인터프리터 실행
`MP`는 쉘처럼 간편하게 사용하도록 설계되었습니다.
때문에 `SSH`를 이용하여 클라이언트가 서버에서 `MP`를 간편히 실행할 수 있습니다.
더 나아가, 일반적인 쉘처럼 사용할 수 있을 뿐만 아니라,
`RemoteInterpreter` 클래스를 이용하여 파이썬 응용 프로그램을 작성할 수 있습니다.
* 원하는 디렉토리에서 작업을 시작하는 부분은 현재 미구현 상태입니다.
#### 쉘을 통한 실행
```bash
python -m mp.remote [hostname] [user] [-p port] [-d dir_process] [-i interpreter]
Password: [password]
```
#### 파이썬을 통한 실행
```python
from mp import RemoteInterpreter

sess = RemoteInterpreter()
sess.connect('[hostname]', '[user]', '[password]', port)
sess.session()
sess.begin_interactive()
```

<a name="ch6-2"></a>
### Anaconda와 같은 가상 환경에서 실행하기
`Python`은 모듈의 의존성이 중요한 프로그래밍 언어입니다.
때문에 모듈 관리에 집중한 다양한 프로그램들이 있습니다.
대표적으로 `Anaconda`가 있는데, 이들 환경에서 `python`을 실행하기 위해선 조금의 과정이 추가됩니다.
`MP`는 `python` 인터프리터를 직접 지정할 수 있게 하여 문제를 해결하려 합니다.
```python
from mp import RemoteInterpreter

sess = RemoteInterpreter()
sess.connect('[hostname]', '[user]', '[password]', port)
sess.session('[interpreter path]')
sess.begin_interactive()
```

## 7. 권장 프로그래밍 습관 (관습)
<a name="ch7-1"></a>
### 변수 이름 짓는 법 (명명법)
`MP`는 비록 의사코드를 지향하지만, 모든 사람들이 이해할 수 있도록 하려면 일종의 관습이 필요합니다.
특히, `MP`는 전역 변수를 기본으로 사용하고, 변수가 곧 파일이기 때문에 특히 중요합니다.
`MP`의 관습 중 가장 기초적인 것은, 정해진 규칙에 따라 변수의 이름을 짓는 것입니다.
`MP`에서 변수의 종류는 다음과 같습니다.

* 일반 변수: 전역 변수
* 매개 변수: 사용자 정의 메소드에 사용된 **일반 변수**는 **매개 변수**가 됩니다.
* 수식: 상수, `tensor()` 등으로 선언된 텐서
* 메소드: `built-in` 메소드
* 사용자 정의 메소드: `def`를 통해 선언한 메소드

여기서, 각 변수는 각자의 **명명 규칙**이 있습니다.

* 일반 변수: 알파벳 소문자, 혹은 이에 준하는 문자로 시작한다.
* 매개 변수: 언더바 `_`로 시작한다.
    * 일반 변수와 충돌되지 않게 한다.
* 수식: 숫자로 시작하며, 숫자 혹은 자료형으로 마친다.
    * 허용되는 자료형
        * 부울 타입: `b`
        * 정수형: byte `i8`, int `i32`, long `i64`
        * 실수형: fixed `f16`, float `f32`, double `f64`
    * 정수형과 실수형을 구분하기 위해, 실수형에는 반드시 점 `.`을 포함하여야 한다.
    * 지수 표기법 `1e-3`을 지원한다.
* 메소드: 두 개의 언더바 `__`로 시작한다.
* 사용자 정의 메소드: 일반 변수와 같다.

또한, 모든 변수들이 지켜야 할 공통의 **명명 규칙**이 있습니다.

* 단어를 결합할 때는 띄워쓰기 ` `를 사용하고, `_`와 `CamelCase`를 지양한다.
    * `my sweet home (O)`, `my_sweet_home (X)`, `mySweetHome (X)`
* 수식을 베끼는 경우, 다음과 같은 접두사를 이용한다.
    * 위첨자: `^`
    * 아래첨자: 따로 붙이지 않는다.
* 변수명이 문장형일 경우, `'`나 `"`와 같은 따옴표를 사용하여 가독성을 높인다.
    * `"Set 'stride' to 2 instead of 1" = true`

<a name="ch7-2"></a>
### 스크립트를 읽기 쉽게 하기 (단락)
`MP`에선 단락을 잘 구분하는 것이 가독성을 높이는 데에 매우 중요합니다.
단락을 잘 구분하기 위해선, 각 단락마다 비슷한 역할을 맡는 문장을 작성해야 합니다.
무엇보다 중요한 것은, 단락을 시작할 땐 이를 잘 설명하는 주석을 다는 것입니다.
그리고 주석과 소스코드를 들여쓰기 (4칸 띄워쓰기)를 통해 구분합니다.
```
# 초기값을 설정합니다.
    a = 3
    b = 4

# 계산을 수행합니다.
    c = a + b

# 계산 결과를 출력합니다.
    print c
```
